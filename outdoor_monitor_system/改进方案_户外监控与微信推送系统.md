# 户外监控系统改进方案
## 基于 VisionFive 2 的低功耗定时监控与实时回传系统（含微信推送）

---

## 一、问题分析

### 原始方案不足
原方案实现了以下功能：
- ✓ 定时抓拍并上传
- ✓ ROI 区域运动检测
- ✓ 事件图片/视频保存
- ✓ LED 状态指示

**缺失的关键功能：**
- ✗ 消息推送到达手机的功能
- ✗ 检测结果通知用户

### 改进目标
实现"**人物经过拍照并向用户手机发送消息**"的完整监控效果。

---

## 二、改进后的系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        VisionFive 2                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐           │

│  │   (定时)     │→ │  (MOG2背景建模)│→ │  (Server酱)  │           │
│  └──────────────┘  └──────────────┘  └──────────────┘           │
│         ↓                  ↓                  ↓                   │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │                     主监控程序                              │   │
│  │  monitor_uploader.py                                      │   │
│  │  - 定时抓拍 → 上传                                          │   │
│  │  - 检测到人 → 拍照 → 上传 → 微信推送 → LED亮起              │   │
│  └──────────────────────────────────────────────────────────┘   │

                              ↓
```

## 三、新增功能模块

### 3.1 运动检测模块 (`motion_detection.py`)

**技术方案：**
- 使用 OpenCV MOG2 背景建模算法
- 对 ROI 区域进行前景检测
- 面积阈值过滤（默认 1200 像素），排除小物体误报
- 支持阴影检测开关

**核心参数：**
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `area_threshold` | 1200 | 最小检测面积（像素） |
| `history` | 500 | 背景建模帧数 |
| `var_threshold` | 25 | 方差阈值 |
| `detect_shadows` | True | 是否检测阴影 |

**检测流程：**
```
当前帧 → 应用 ROI 掩码 → MOG2 背景建模 → 开闭运算去噪 → 轮廓提取 → 面积过滤 → 输出结果
```

### 3.2 微信推送模块 (`push_notification.py`)

**技术方案：**
- 使用 Server酱（sc.ftqq.com）微信推送服务
- 调用 HTTP API 发送消息
- 支持 Markdown 格式

**使用步骤：**
1. 访问 https://sc.ftqq.com/ 注册账号
2. 扫码关注"方糖"公众号，绑定微信
3. 获取 SCKEY

**推送内容格式：**
```
标题: 🚨 监控报警 [VF2-01]

## 检测到人员入侵

- **设备ID**: VF2-01
- **检测时间**: 2026-04-28 15:30:45
- **位置**: 监控区域
- **检测目标数**: 1 个

**事件图片**: [点击查看](http://服务器IP:5000/uploads/xxx.jpg)
```

### 3.3 主监控程序 (`monitor_uploader.py`)

**工作流程：**
```
启动
  ↓
初始化摄像头、GPIO、检测器、推送器
  ↓
┌─────────────────────────────────────┐
│         监控周期循环                  │
├─────────────────────────────────────┤
│ 1. 前 2 秒内抓拍定时图并上传          │
│                                     │
│ 2. 运动检测循环（持续 work-sec 秒）   │
│    ├── 读取帧                        │
│    ├── 背景建模检测                   │
│    ├── 是否检测到运动？               │
│    │   ├── 否 → 继续检测              │
│    │   └── 是 →                      │
│    │       ① 保存事件图               │
│    │       ② 上传到服务器            │
│    │       ③ 发送微信推送            │
│    │       ④ 开始录像                │
│    │       ⑤ LED 亮起               │
│    │       ⑥ 等待录像完成            │
│    │       ⑦ LED 关闭                │
│    └── 按 q 可退出                    │
└─────────────────────────────────────┘
  ↓
停止监控，清理资源
```

**命令行参数：**
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--camera-index` | 4 | 摄像头设备号 |
| `--device-id` | VF2-01 | 设备标识 |
| `--upload-url` | http://... | 上传地址 |
| `--work-sec` | 20 | 单次任务运行时长 |
| `--record-sec` | 10 | 事件录像时长 |
| `--sckey` | - | Server酱 SCKEY |
| `--no-push` | False | 禁用推送 |
| `--detect-mode` | 1 | 0=仅抓拍, 1=抓拍+检测 |

## 四、服务器端 (`server_receive.py`)

### 功能
- 接收监控设备上传的文件
- 提供 Web 文件浏览界面
- 提供图片访问 URL（供微信推送链接使用）

### API 接口

| 路由 | 方法 | 说明 |
|------|------|------|
| `/upload` | POST | 接收文件上传 |
| `/uploads/<file>` | GET | 访问上传的文件 |
| `/` | GET | Web 管理界面 |
| `/api/status` | GET | 服务器状态 |
| `/health` | GET | 健康检查 |

### 启动方式
```bash
python3 server_receive.py
```

## 五、硬件接线（与原方案一致）

### LED 指示灯
- GPIO36（物理引脚 36）→ 串联 470Ω 电阻 → LED 正极
- LED 负极 → GND（物理引脚 34）

### USB 摄像头
- 直接插入 VisionFive 2 的 USB Host 接口

### 供电
- 5V / 3A USB-C 电源适配器

## 六、systemd 定时任务（可选）

如果需要周期性地执行监控任务（非真正低功耗版）：

**服务文件** `outdoor-monitor.service`：
```ini
[Unit]
Description=Outdoor timed monitor with push notification
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
WorkingDirectory=/home/user/outdoor_monitor
ExecStart=/usr/bin/python3 /home/user/outdoor_monitor/monitor_uploader.py \
    --camera-index 4 \
    --device-id vf2-01 \
    --upload-url http://你的服务器IP:5000/upload \
    --sckey 你的SCKEY \
    --work-sec 20 \
    --record-sec 10
```

**定时器文件** `outdoor-monitor.timer`：
```ini
[Unit]
Description=Run outdoor monitor every 5 minutes

[Timer]
OnBootSec=1min
OnUnitActiveSec=5min
AccuracySec=10s
Persistent=true

[Install]
WantedBy=timers.target
```

**启用定时任务：**
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now outdoor-monitor.timer
systemctl list-timers | grep outdoor-monitor
```

## 七、完整运行命令

### 1. 服务器端
```bash
# 在有公网 IP 或内网穿透的服务器上运行
python3 server_receive.py
```

### 2. VisionFive 2 监控端
```bash
cd /home/user/outdoor_monitor

# 测试摄像头
python3 -c "
import cv2
cap = cv2.VideoCapture(4)
ret, frame = cap.read()
print('OK' if ret else 'FAIL')
cap.release()
"

# 运行监控程序
python3 monitor_uploader.py \
    --camera-index 4 \
    --device-id VF2-01 \
    --upload-url http://服务器IP:5000/upload \
    --sckey 你的SCKEY \
    --work-sec 20 \
    --record-sec 10

# 或启用 systemd 定时任务
sudo cp outdoor-monitor.service /etc/systemd/system/
sudo cp outdoor-monitor.timer /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now outdoor-monitor.timer
```

## 八、预期效果

运行程序后，当有人经过摄像头监控区域时：

1. **本地**：保存事件图片到 `snapshots/` 目录
2. **上传**：事件图片自动上传到 Flask 服务器
3. **推送**：你的微信会收到如下通知：
   ```
   🚨 监控报警 [VF2-01]

   检测到人员入侵
   设备ID: VF2-01
   检测时间: 2026-04-28 15:30:45
   位置: 监控区域
   检测目标数: 1 个

   事件图片: 点击查看
   ```
4. **LED**：状态灯亮起（检测到目标时）
5. **录像**：自动录制 10 秒事件视频

## 十、低功耗扩展（下一步）

当前版本属于"常开版定时监控"，若课程要求真正低功耗：

```
RTC/PIR → 低功耗MCU → 5V电源开关 → VisionFive 2
                                    ↓
                            任务完成 → DONE信号
                                    ↓
                            MCU切断电源
```

平时 MCU 深度休眠，仅 RTC 或 PIR 工作；检测到人或到达定时时间时才上电，任务完成后自动断电。
